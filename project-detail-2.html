Python Data
Transformation (TA04)

Project Overview

This project, titled "Data Management and Transformation with Python (TA04)", focuses on automating the data flow between industry-standard formats (CSV, XML, JSON). It was developed within the academic context of the Institut Tecnològic de Barcelona.

The main objective was to build efficient scripts for the migration, validation, and analysis of information regarding IT equipment incidents, covering the complete data lifecycle from ingestion to persistence.

The Workflow

The process began with Data Collection, creating a system to register hardware and software incidents and exporting initial results to CSV. I then developed an ETL (Extract, Transform, Load) pipeline using Python to convert this raw data into XML.

A key challenge was ensuring XML tag validity using Regular Expressions (Regex) and strictly adhering to GDPR privacy standards. Finally, the system analyzes the XML to filter errors and exports the processed results to JSON, ensuring data persistence and future reusability.

Key Functionalities

The system handles Data Ingestion by standardizing form inputs into CSV format. It features a custom CSV-to-XML Migration algorithm that cleans keys and validates tags dynamically.

Additionally, the tool performs Processing & Analysis by parsing XML files to filter erroneous data (such as invalid dates) and generating incident statistics via the console. The final step involves JSON Export, implementing file existence checks to safely store the processed information.

Technical Specifications

The project relies heavily on Python and its core libraries (csv, xml.etree.ElementTree, re) to handle logic and file manipulation without heavy external dependencies. Documentation was maintained using Markdown.

Key competencies demonstrated include advanced Scripting, the use of Regular Expressions (Regex) for data validation, and the implementation of GDPR principles to ensure data privacy during the transformation process.
