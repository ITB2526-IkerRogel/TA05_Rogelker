Red Team Operations

Advanced Corporate
Infrastructure Pentest

Audit Objectives & Scope

This operation simulated a targeted attack against "TechCorp", a fictitious entity with a complex segmented network. The primary objective was to demonstrate the business impact of unpatched vulnerabilities and weak configurations. The engagement followed a Black Box approach, starting with zero knowledge.

The initial phase involved passive reconnaissance and active port scanning using Nmap and Masscan to map the attack surface and identify outdated services running on the perimeter.

Exploitation & Initial Access

In the exploitation phase, we identified a legacy web server vulnerable to deserialization attacks. Metasploit Framework was used to leverage this CVE, allowing us to deploy a reverse shell and gain initial access to the DMZ.

Once inside, we established persistence using scheduled tasks and set up a C2 (Command & Control) channel over encrypted HTTPS traffic to evade basic firewall detection rules.

Active Directory Lateral Movement

After pivoting to the internal network using Chisel, we utilized BloodHound to map the Active Directory trust relationships. We identified a path to Domain Admin via a user with excessive delegation rights.

We performed a Kerberoasting attack to extract service account hashes and cracked them offline using Hashcat. This allowed us to impersonate a privileged user and move laterally to the Domain Controller.

Results & Remediation

The final result was a detailed technical report with mitigation proposals (Hardening) and firewall configuration updates to prevent future attacks and ensure data integrity.

Key recommendations included enforcing MFA for all remote access, patching the deserialization vulnerability, and adhering to the principle of least privilege for service accounts within the Active Directory.
