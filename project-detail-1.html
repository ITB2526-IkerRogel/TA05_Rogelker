SOC Automation & Threat Hunting

This project involved building a fully functional Security Operations Center (SOC) from scratch using open-source technologies. The goal was to create a centralized platform capable of ingesting logs from firewalls, servers, and endpoints to detect anomalies in real-time.

By integrating Wazuh agents for endpoint telemetry and the ELK Stack for indexing and visualization, I established a robust pipeline for security events. The system filters noise and highlights high-fidelity alerts mapped to the MITRE ATT&CK framework.

Automation & Incident Response

Automation was achieved using TheHive and Cortex to auto-analyze suspicious IPs and file hashes, significantly reducing the Mean Time to Detect (MTTD). Custom Sigma rules were developed and converted to Elastic queries to detect specific lateral movement techniques.

We configured active response scripts in Wazuh to automatically block IP addresses performing brute-force attacks on SSH and RDP services, providing an immediate layer of defense without human intervention.

Challenges & Solutions

One of the main challenges was handling the high volume of logs generated by the Windows Event Forwarding system. To solve this, we implemented a Kafka message queue buffer before Logstash, ensuring no data loss during traffic spikes.

Another hurdle was false positives in the IDS. We spent two weeks fine-tuning the Suricata ruleset, creating a whitelist for authorized internal administrative traffic, which improved the alert accuracy by 40%.

Final Results & Impact

The final deliverable included comprehensive Kibana dashboards for visualizing attack trends and a fully documented incident response playbook. The system successfully detected a simulated red team attack during the validation phase.

This architecture provides a scalable foundation for any organization looking to implement 24/7 monitoring with zero licensing costs, proving that open-source security can rival enterprise solutions.
